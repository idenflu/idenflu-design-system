Components

Attachments

파일 상태, progress, 실패 복구를 업무 흐름 안에서 보여주는 기준입니다.

Component standard

Attachments 문서의 공통 구조.

Anatomy

File input, file row, thumbnail, progress, error, action을 나눕니다.

  • Upload trigger
  • file name
  • size
  • progress
  • remove action
Variants

Upload zone, file row, preview item, failed upload, scanning item

States

Empty, uploading, complete, failed, scanning, disabled 상태를 비교합니다.

Accessibility

File input label, progress text, error reason, remove aria-label을 확인합니다.

Examples

Asset upload, brief file, report export, approval attachment에 사용합니다.

QA

긴 파일명, 실패 복구, progress text, mobile row stack을 확인합니다.

Document template

Attachments 문서는 파일 상태와 다음 action을 함께 보여줍니다.

01 Overview

업로드인지, 이미 첨부된 파일 목록인지 먼저 구분합니다.

02 Anatomy

Trigger, file row, thumbnail, metadata, progress, action을 분리합니다.

03 Variants

Upload zone, file list, preview, failed row, scanning row를 구분합니다.

04 States

Empty, uploading, complete, failed, disabled 상태를 비교합니다.

05 Accessibility

파일 input label, progress announcement, remove label을 확인합니다.

06 QA

긴 파일명, 큰 파일, 실패 복구, mobile overflow를 검증합니다.

Attachment는 파일 상태를 row 단위로 추적합니다.

Review checklist
  • 파일명, 크기, 상태를 같은 row 안에 표시합니다.
  • 실패 row에는 원인과 복구 action을 제공합니다.
  • 진행률은 bar만 쓰지 않고 숫자 text를 함께 둡니다.

Token contract

Attachments는 row, progress, error token을 함께 씁니다.

DecisionTokenRule
Row text{typography.body-sm}파일 metadata는 compact row와 같은 텍스트 기준을 씁니다.
Surface{colors.surface-raised}파일 row는 hairline으로만 구분합니다.
Error{colors.accent-coral}실패 row는 coral과 retry action을 함께 둡니다.
Spacing{spacing.md}파일 아이콘, metadata, action 간격은 16px 기준입니다.

API contract

Attachment API 구현 계약.

TypeContract
Propsfile name, file type, size, progress, preview, action
Statesempty, uploading, complete, failed, scanning, disabled
A11yfile input label, progress text, error reason, remove action aria-label

Token usage

Attachment tokens를 실제 token path에 연결합니다.

{typography.body-sm} {colors.surface-raised} {colors.hairline} {colors.accent-coral} {spacing.md}

Usage guidance

idenflu 화면에서 쓰는 기준을 먼저 정합니다.

Use

brief, proof, report처럼 파일의 상태, 크기, progress, 실패 복구를 보여줄 때 씁니다.

Avoid

업로드 실패나 scanning 상태를 파일명만 바꿔 표현하지 않습니다.

Replace with

단순 링크는 button/link, 이미지 검토는 preview card, 실패 안내는 feedback을 함께 씁니다.
