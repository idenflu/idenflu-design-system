Components

Badges

Status labels, count badges, filter chips, and removable tags.

Component standard

Badges 문서의 공통 구조.

Anatomy

Tone, label, count, optional icon, remove action을 나눕니다.

  • Text label
  • tone
  • count
  • remove button
  • status meaning
Variants

Status badge, count badge, filter chip, removable tag, risk label

States

Neutral, info, success, warning, danger, selected, disabled 상태를 비교합니다.

Accessibility

상태는 색상만으로 전달하지 않고 remove action은 aria-label을 사용합니다.

Examples

테이블 상태, 필터 적용값, owner label, asset risk에 사용합니다.

QA

긴 label, count overflow, disabled chip, dark contrast를 확인합니다.

Context example

Badges는 상태와 적용된 조건을 작은 단위로 보여줍니다.

Selected Review pending Approved Disclosure risk
Review checklist
  • Status badge는 상태 text를 반드시 포함합니다.
  • Filter chip은 선택 조건과 reset 가능 여부를 드러냅니다.
  • Count badge는 단독 의미가 없으면 label 옆에 붙입니다.

Token contract

Badges는 caption과 semantic accent를 작은 면적으로 사용합니다.

DecisionTokenRule
Text{typography.caption}Badge label은 12px caption으로 유지합니다.
Info{colors.primary-soft}선택과 정보는 primary soft를 사용합니다.
Success{colors.accent-mint}완료 상태는 mint와 완료 label을 함께 씁니다.
Warning{colors.accent-amber}대기와 위험 예고는 amber와 원인 text를 함께 둡니다.
