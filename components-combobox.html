Component standard

Combobox 문서의 공통 구조.

Anatomy

Label, input, popup, option, active option, selection을 나눕니다.

  • Visible label
  • query input
  • option list
  • active option
  • helper copy
Variants

Single select, multi select, async search, owner picker, tag picker

States

Collapsed, expanded, typing, active, empty, loading, disabled 상태를 비교합니다.

Accessibility

role combobox, aria-controls, aria-activedescendant, option keyboard path를 확인합니다.

Review checklist
  • Active option은 query input과 aria-activedescendant로 연결합니다.
  • 검색 결과가 없으면 원인과 reset action을 제공합니다.
  • 서버 검색은 loading text를 option 영역 안에 표시합니다.

Token contract

Combobox는 field와 menu surface 토큰을 함께 씁니다.

DecisionTokenRule
Input{sizing.field-height}입력 영역은 field와 같은 높이를 사용합니다.
Popup{colors.surface-raised}옵션 영역은 surface-raised와 hairline으로 구분합니다.
Active{colors.primary-soft}키보드 active option은 selected와 구분되는 soft surface를 씁니다.
Focus{colors.primary}입력 focus와 option active가 동시에 읽히도록 유지합니다.
