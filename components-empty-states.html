Empty states

빈 결과, 오류, 권한 제한에서도 다음 행동을 제공하는 기준입니다.

Component standard

Empty states 문서의 공통 구조.

Anatomy

Title, reason, recovery action, secondary action, permission reason을 나눕니다.

  • Title
  • reason
  • primary action
  • secondary action
  • permission reason
Variants

Empty, filtered empty, error, permission, locked, loading recovery

States

Empty, filtered empty, error, permission, locked, loading 상태를 비교합니다.

Accessibility

Clear heading, recovery action, alert role, permission reason visible을 확인합니다.

Examples

No campaigns, filtered creator list, sync error, owner-only approval에 사용합니다.

QA

원인 없는 empty, action 없는 error, 권한 사유 누락을 확인합니다.

Document template

Empty states 문서는 원인과 복구 action을 함께 제공합니다.

01 Overview

빈 상태인지, 오류인지, 권한 문제인지 먼저 구분합니다.

02 Anatomy

Title, reason, primary action, secondary action, helper를 분리합니다.

03 Variants

04 States

Loading, retry, permission, locked, filtered 상태를 비교합니다.

05 Accessibility

Heading, role alert, action label, permission reason을 확인합니다.

06 QA

긴 reason, mobile stack, action 없는 상태, dark contrast를 검증합니다.

Context example

상태 surface는 왜 막혔는지와 다음 행동을 명확히 제공합니다.

No matching creators

Clear one filter or broaden audience fit.

Sync failed

Retry keeps selected rows and filters.

Owner approval required

Permission recovery is available after Mina approves Rev 03.

Review checklist
  • Filtered empty는 일반 empty와 다른 복구 action을 제공합니다.
  • Error는 role alert와 retry 보존 범위를 함께 설명합니다.
  • Permission 상태는 누가, 왜, 어떻게 풀 수 있는지 표시합니다.

Token contract

Empty states는 제목, 이유, 복구 action의 계층을 유지합니다.

DecisionTokenRule
Title{typography.card-title}상태 제목은 문제를 한 문장으로 설명합니다.
Reason{typography.body-sm}원인과 결과는 body-sm로 짧게 제공합니다.
Action{colors.primary}복구 action은 primary 또는 ghost action으로 제공합니다.
Error{colors.accent-coral}오류는 coral과 role alert를 함께 씁니다.

Usage guidance

idenflu 화면에서 쓰는 기준을 먼저 정합니다.

Use

true empty, filtered empty, permission empty, error empty처럼 다음 행동이 필요한 빈 화면에 씁니다.

Avoid

모든 빈 화면에 같은 Create CTA를 반복하지 않습니다.

Replace with

일시 로딩은 skeleton/progress, inline 오류는 feedback, 권한 안내는 locked panel과 연결합니다.
