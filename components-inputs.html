Inputs

Fields, selects, validation, decoration, disabled states.

Component standard

Inputs 문서의 공통 구조.

Anatomy

기본 구성요소를 정리해 구현과 디자인 리뷰의 출발점을 맞춥니다.

  • Visible label
  • control
  • helper text
  • prefix/suffix
  • validation state
Variants

Text, search, select, textarea, decorated, readonly, disabled

States

Default, hover, focus, selected, loading, disabled, error 또는 locked 상태를 같은 density로 비교합니다.

Accessibility

Keyboard path, visible focus, semantic role, accessible name, helper/error 연결을 함께 검토합니다.

Examples

단독 컴포넌트보다 campaign, creator, review, admin workflow 안에서 쓰이는 예시를 우선합니다.

QA

Visual QA에서 light/dark, compact density, disabled hover, focus, long text overflow를 다시 확인합니다.

Form flows

입력 컴포넌트가 실제 업무 흐름에서 보이는 방식.

Multi-step setup

Campaign setup은 brief, creators, review rule처럼 단계별 완료 상태를 명확히 보여줍니다.

  • 현재 step은 strong label과 primary action으로 강조합니다.
  • 완료 step은 check icon과 수정 action을 제공합니다.
  • 막힌 step은 blocked reason을 field helper로 둡니다.
Autosave dirty state

운영자가 긴 brief를 수정할 때 저장 상태는 toolbar와 field 근처에 동시에 읽혀야 합니다.

  • Dirty: Unsaved changes
  • Saving: aria-busy와 inline loading
  • Saved: timestamp와 owner context
Server validation

클라이언트 검증 후에도 서버에서 거절될 수 있는 rule은 복구 action과 함께 보여줍니다.

  • 에러 field는 aria-invalid를 사용합니다.
  • 상단 summary는 문제가 있는 field로 연결합니다.
  • 재시도는 같은 값을 유지한 채 실행합니다.
Permission lock

권한 때문에 편집이 불가능한 field는 disabled가 아니라 readonly + lock reason이 더 적합할 수 있습니다.

  • 복사가 필요한 ID와 URL은 readonly입니다.
  • action 자체가 불가능하면 disabled입니다.
  • 권한 요청 action은 helper 아래에 배치합니다.
Decorated finance fields

금액, 통화, 단위, timezone은 value 바깥의 prefix/suffix로 처리합니다.

  • 입력값은 항상 가장 높은 대비를 유지합니다.
  • prefix/suffix는 muted text로 둡니다.
  • 장식 아이콘은 focus target이 아닙니다.
Command search

검색형 input은 입력, 추천 결과, keyboard hint가 같은 control group처럼 보여야 합니다.

  • Leading search icon
  • Trailing shortcut hint
  • Combobox active descendant
