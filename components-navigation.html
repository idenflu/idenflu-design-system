Component standard

Navigation 문서의 공통 구조.

Anatomy

기본 구성요소를 정리해 구현과 디자인 리뷰의 출발점을 맞춥니다.

  • Landmark
  • active item
  • group label
  • shortcut
  • collapse state
Variants

Top nav, side nav, breadcrumb, pagination, command jump

States

Default, hover, focus, selected, loading, disabled, error 또는 locked 상태를 같은 density로 비교합니다.

Accessibility

Keyboard path, visible focus, semantic role, accessible name, helper/error 연결을 함께 검토합니다.

Examples

단독 컴포넌트보다 idenflu 제품 화면 안에서 쓰이는 예시를 우선합니다.

QA

Visual QA에서 light/dark, compact density, disabled hover, focus, long text overflow를 다시 확인합니다.

Review checklist
  • 현재 위치는 active 또는 aria-current로 명시합니다.
  • Toolbar는 이동이 아니라 현재 화면의 작업 도구입니다.
  • Pagination은 전체 범위와 현재 페이지를 함께 말해야 합니다.

Token contract

Navigation은 density와 active state 토큰으로 반복 이동을 안정화합니다.

DecisionTokenRule
Active{colors.primary-soft}현재 위치는 배경과 primary text를 함께 사용합니다.
Toolbar{density.compact}반복 작업 도구는 compact height와 명확한 label을 유지합니다.
Border{colors.hairline}Navigation chrome은 hairline으로 영역을 구분합니다.
Focus{colors.primary}키보드 이동 경로가 active state와 구분되어야 합니다.

Navigation API 구현 계약.

TypeContract
