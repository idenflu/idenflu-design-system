Overlay states

Open 상태만 만들지 말고 pending, validation, locked까지 정의합니다.

Pending action

Assign, save, reject처럼 네트워크가 필요한 action은 버튼 label과 loading state를 유지합니다.

  • aria-busy 사용
  • 중복 submit 방지
  • 취소 가능 여부를 명확히 표현
Validation

Popover나 modal 안의 field error는 overlay 안에서 해결되어야 합니다.

  • 첫 error로 focus 이동
  • aria-describedby 연결
  • 값 보존 후 retry
Locked overlay

권한이 없어 action이 불가능하면 dialog를 열기 전에 disabled + reason을 우선합니다.

  • 권한 요청 action 제공
  • owner 또는 policy 노출
  • 숨겨진 destructive action은 피합니다

Focus restore

닫힌 뒤 사용자가 있던 자리로 돌아와야 합니다.

Open
  1. Trigger stores restore target.
  2. aria-expanded becomes true.
  3. Overlay becomes visible.
  4. First safe focus target receives focus.
Navigate
  1. Tab order stays inside modal or drawer.
  2. Esc closes when it is non-destructive.
  3. Menu arrow keys move active menuitem.
  4. Validation sends focus to the first invalid field.
Close
  1. Overlay hidden state is restored.
  2. aria-expanded becomes false.
  3. Focus returns to trigger.
  4. Selection/filter context remains unchanged.
