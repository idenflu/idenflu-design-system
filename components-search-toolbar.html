Components

Search toolbar

검색, 필터, saved view, reset, density 조작을 한 흐름으로 묶는 기준입니다.

Component standard

Search toolbar 문서의 공통 구조.

Anatomy

Search field, filter trigger, active chips, saved view, reset, density action을 나눕니다.

  • Search field
  • filter triggers
  • active chips
  • saved view
  • reset action
Variants

Table toolbar, list toolbar, saved view toolbar, compact mobile toolbar

States

Empty query, typing, filtered, saved, loading, disabled 상태를 비교합니다.

Accessibility

Search label, filter button name, active filter text, reset action을 확인합니다.

Examples

Campaign table, review queue, creator shortlist 상단에서 사용합니다.

QA

긴 query, 많은 filter chip, mobile wrap, disabled reset을 확인합니다.

Context example

Search toolbar는 목록의 현재 조건을 숨기지 않고 보여줍니다.

Review checklist
  • 적용된 필터는 chip으로 직접 노출합니다.
  • 검색 입력과 필터 button은 같은 keyboard 흐름에 둡니다.
  • 조건이 없으면 reset action은 disabled 처리합니다.

Token contract

Search toolbar는 field, chip, compact button 토큰을 함께 씁니다.

DecisionTokenRule
Search field{sizing.field-height}검색 입력은 field height를 유지합니다.
Density{density.compact}테이블 상단에서는 compact gap과 button size를 사용합니다.
Surface{colors.surface-raised}Toolbar는 표면을 만들되 nested card처럼 보이지 않게 합니다.
Filter chip{colors.primary-soft}적용된 조건은 primary soft와 count로 표시합니다.

Usage guidance

idenflu 화면에서 쓰는 기준을 먼저 정합니다.

Use

테이블이나 리스트를 검색, 필터, saved view, density 기준으로 반복 조작할 때 씁니다.

Avoid

검색과 action, 설정을 구분 없이 한 줄에 계속 추가하지 않습니다.

Replace with

단일 검색은 input, 소수 필터는 controls, 세부 설정은 overlay나 menu로 분리합니다.
