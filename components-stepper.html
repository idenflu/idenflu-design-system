Components

Stepper

Setup, review, approval progress, and blocked steps.

Component standard

Stepper 문서의 공통 구조.

Anatomy

Step label, order, status, helper, action, blocked reason을 나눕니다.

  • Step number
  • label
  • status
  • helper
  • blocked reason
Variants

Setup stepper, approval flow, horizontal progress, compact checklist

States

Not started, current, complete, blocked, optional, error 상태를 비교합니다.

Accessibility

Ordered list, aria-current step, visible status, blocked reason을 확인합니다.

Examples

Campaign setup, review approval, export workflow, permission setup에 사용합니다.

QA

긴 label, blocked reason, optional step, mobile stack을 확인합니다.

Stepper 문서는 현재 단계와 막힌 이유를 함께 보여줍니다.

01Overview

단계가 순차적인지, 단순 checklist인지 먼저 구분합니다.

02Anatomy

Order, label, state, helper, action, blocked reason을 분리합니다.

Setup, approval, checklist, compact progress를 구분합니다.

04States

Current, complete, blocked, optional, error 상태를 비교합니다.

05Accessibility

aria-current, ordered list, status text, action label을 확인합니다.

06QA

Mobile stack, long text, blocked reason, completed contrast를 검증합니다.

Context example

Review checklist
  • 현재 step은 aria-current와 visible label을 함께 씁니다.
  • Blocked step은 원인과 다음 action을 제공합니다.
  • Optional step은 건너뛸 수 있음을 text로 표시합니다.

Token contract

Stepper는 진행 상태를 색상과 텍스트로 함께 표현합니다.

DecisionTokenRule
Label{typography.body-emphasis}Step label은 강조 텍스트로 스캔되게 합니다.
Current{colors.primary}현재 단계는 primary와 aria-current를 함께 씁니다.
Complete{colors.accent-mint}완료 단계는 mint와 완료 text를 함께 둡니다.
Divider{colors.hairline}단계 연결선은 hairline 수준으로만 표시합니다.
