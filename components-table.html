Advanced table patterns

운영 테이블에서 자주 필요한 상태.

Sticky header

긴 roster나 invoice 테이블은 header와 bulk selection bar를 같은 스크롤 맥락에 고정합니다.

  • Header cell은 columnheader를 유지합니다.
  • Active sort는 aria-sort로 노출합니다.
  • 가로 스크롤에서도 row action 컬럼이 사라지지 않게 설계합니다.
Column resize

Campaign name, creator, note처럼 길이가 크게 변하는 컬럼에만 resize affordance를 둡니다.

  • Resize handle은 header 오른쪽 8px hit area를 둡니다.
  • 최소 폭 이하로 줄어들면 text truncation을 사용합니다.
  • 저장된 width는 workspace preference로 유지합니다.
Expanded row

행을 열었을 때는 별도 카드가 아니라 같은 table context 안에서 audit, notes, blockers를 보여줍니다.

  • Trigger는 aria-expanded를 사용합니다.
  • Expanded content는 row 아래 full-span 영역입니다.
  • 닫아도 selection과 filters는 유지합니다.
Filtered empty

필터 결과가 없는 상태와 데이터 자체가 없는 상태를 분리합니다.

  • 적용된 filter summary를 보여줍니다.
  • Clear filter를 quiet action으로 제공합니다.
  • 새 항목 생성 CTA는 진짜 empty일 때만 primary로 둡니다.
Permission locked row

권한 때문에 불가능한 action은 숨기지 말고 disabled 상태와 사유를 함께 보여줍니다.

  • 잠긴 row action은 disabled button을 사용합니다.
  • 사유는 row detail 또는 helper copy로 노출합니다.
  • 권한 요청 action은 별도 quiet action으로 둡니다.
Density switch

Compact, comfortable, spacious는 사용자 취향이 아니라 작업 유형에 따라 기본값을 정합니다.

  • Review queue는 compact를 기본으로 둡니다.
  • Campaign detail은 comfortable을 기본으로 둡니다.
  • Approval modal은 spacious를 선택합니다.

Table behavior blueprint

정렬, 필터, 선택, 확장, 모바일 fallback까지 같은 모델로 처리합니다.

Toolbar model

Toolbar는 title, updated time, filter controls, density, export를 포함합니다.

  • Filter summary는 table 밖이 아니라 toolbar에 유지합니다.
  • Export는 현재 filter와 selected scope를 명시합니다.
  • Density는 저장 가능한 user preference입니다.
Bulk action model

Selection bar는 selected count, affected scope, available actions를 함께 보여줍니다.

  • Visible rows only와 all filtered rows를 구분합니다.
  • 권한 때문에 일부 row가 제외되면 제외 사유를 표시합니다.
  • Clear selection은 항상 ghost action입니다.
Responsive fallback

작은 화면에서는 table을 억지로 압축하지 않고 row card로 전환합니다.

  • Title, status, owner, revision, due, risk는 유지합니다.
  • Row action은 card footer에서 계속 접근 가능합니다.
  • 비교가 핵심이면 horizontal scroll을 허용합니다.
StateRequired UIRecovery or action
