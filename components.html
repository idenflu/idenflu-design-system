Components

Components

컴포넌트 카탈로그는 인덱스로 유지하고, 세부 문서는 컴포넌트별 페이지에서 관리합니다.

Component catalog

컴포넌트별 상세 문서로 분리했습니다.

각 페이지는 Overview, variants, states, accessibility, examples를 확장하기 위한 독립 문서입니다.

Core components
Overview 컴포넌트 카탈로그와 공통 문서 계약입니다. Buttons 운영 action의 위계, 크기, 로딩, 비활성 기준입니다. Icons 작업 의미를 보조하는 SVG symbol과 접근성 기준입니다. Cards 캠페인, creator, metric, activity 객체 요약 기준입니다. Avatar owner, reviewer, creator 식별과 fallback 기준입니다.
Input and choice
Inputs 값 입력, 검증, helper, disabled/readonly 기준입니다. Controls 필터, 설정, 선택 상태를 유지하는 control 기준입니다. Tabs 같은 업무 맥락 안에서 view mode를 전환하는 기준입니다. Menu 보조 명령, 정렬, 필터, 위험 명령을 묶는 기준입니다. Combobox 검색 가능한 owner/creator/label 선택 기준입니다. Badges 상태, count, filter chip, removable tag 기준입니다. Tooltips 짧은 보조 설명과 metadata popover 기준입니다.
Data and workflow
Search toolbar 검색, 필터, saved view, reset, density 조작 기준입니다. Data table 비교, 정렬, 선택, row action, pagination 기준입니다. Attachments 파일 업로드, progress, 실패 복구, preview 기준입니다. Date time due date, campaign period, timezone, validation 기준입니다. Stepper brief, review, approval 단계와 blocked reason 기준입니다. Empty states 빈 결과, 오류, 권한, 복구 action 기준입니다.
System surfaces
Examples idenflu 운영 화면 조합과 실패/권한 상태 기준입니다. Feedback notice, progress, toast, 오류 복구 커뮤니케이션 기준입니다. Overlays modal, sheet, popover 결정 표면 기준입니다. Navigation 페이지, 섹션, pagination 이동 기준입니다.

Component API

핵심 컴포넌트는 props, states, accessibility 계약을 함께 봅니다.

component-api.json을 기준으로 Button, Input, Table, Overlay 문서의 구현 계약을 한곳에서 비교합니다.

Button API
Props
variant, size, icon, loading, disabled
States
default, hover, focus, selected, loading, disabled, danger
Accessibility
button element, aria-busy for loading, aria-label for icon-only
Icon API
Props
symbol, size, stroke, decorative, label
States
decorative, informative, action, disabled, selected
Accessibility
aria-hidden for decoration, aria-label for icon-only action, stable symbol id
Input API
Props
type, label, helper, validation, availability
States
default, focus, filled, invalid, success, readonly, disabled
Accessibility
visible label, aria-invalid, aria-describedby, combobox relation when searchable
Control API
Props
label, value, checked, group, helper
States
checked, selected, indeterminate, disabled, focus
Accessibility
native input, group label, aria-checked where needed, keyboard path
Tab API
Props
tabs, selected, panel, disabled, overflow
States
selected, hover, focus, disabled, loading panel
Accessibility
role tablist, aria-selected, aria-controls, arrow key navigation
Card API
Props
header, body, footer, metadata, action
States
default, hover, selected, loading, locked, error
Accessibility
card action separation, visible metadata, focus target, semantic heading
Table API
Props
density, selection, sorting, row state, responsive
States
default, selected, expanded, loading, error, locked
Accessibility
role table, columnheader, aria-sort, row action without hover
Overlay API
Props
trigger, surface, backdrop, placement, close rule
States
closed, open, pending, validation, locked, destructive
Accessibility
role dialog, aria-modal, aria-labelledby, focus trap, focus restore
Feedback API
Props
tone, message, reason, action, live region
States
info, success, warning, error, loading, locked
Accessibility
role status or alert, visible reason, recovery action label
Navigation API
Props
landmark, items, current, count, pagination
States
active, hover, focus, disabled, collapsed, current page
Accessibility
nav landmark, aria-current, toolbar label, pagination label
Example Composition API
Props
scenario, component mix, metadata, decision action, recovery state
States
default, review, blocked, permission, empty, error
Accessibility
visible scenario heading, keyboard order follows screen flow, recovery action label
Menu API
Props
trigger, items, group, checked item, danger item
States
closed, open, hover, focus, checked, disabled, danger
Accessibility
aria-haspopup, aria-expanded, role menu, menuitem, Escape closes
Combobox API
Props
label, query, options, active option, selection
States
collapsed, expanded, typing, active, empty, loading, disabled
Accessibility
role combobox, aria-controls, aria-activedescendant, keyboard option navigation
Badge API
Props
tone, label, count, removable, status
States
neutral, info, success, warning, danger, selected, disabled
Accessibility
text visible, status not color-only, remove button aria-label
Tooltip API
Props
trigger, content, placement, interactive, delay
States
hidden, visible, hover, focus, pinned, disabled
Accessibility
aria-describedby, focus trigger, no critical action inside tooltip, popover close path
Avatar API
Props
name, image, initials, size, presence, subtitle
States
image, initials, unassigned, online, busy, disabled
Accessibility
img alt when informative, decorative avatar hidden, visible name near identity rows
Search Toolbar API
Props
query, filters, saved view, density, reset
States
empty query, typing, filtered, saved, loading, disabled
Accessibility
search landmark or label, filter button names, active filter text, reset action
Attachment API
Props
file name, file type, size, progress, preview, action
States
empty, uploading, complete, failed, scanning, disabled
Accessibility
file input label, progress text, error reason, remove action aria-label
Date Time API
Props
value, timezone, range, min date, validation
States
empty, selected, range, disabled date, invalid, readonly
Accessibility
visible label, timezone text, aria-invalid, keyboard date navigation
Stepper API
Props
steps, current step, status, optional, action
States
not started, current, complete, blocked, optional, error
Accessibility
ordered list, aria-current step, visible status, blocked reason
Empty State API
Props
tone, title, reason, primary action, secondary action
States
empty, filtered empty, error, permission, locked, loading
Accessibility
clear heading, recovery action, alert role for errors, permission reason visible

Component detail template

모든 상세 페이지는 같은 깊이로 판단합니다.

컴포넌트는 예쁜 샘플보다 구현 계약, 상태, 접근성, 실제 화면 검증까지 함께 문서화되어야 합니다.

01 Required depth

Anatomy, variants, states, accessibility, examples, QA를 같은 순서로 확인합니다.

02 State model

Default, hover, focus, selected, loading, disabled, error, locked를 빠뜨리지 않습니다.

03 A11y contract

Keyboard path, accessible name, semantic role, focus restore를 구현 전 기준으로 둡니다.

04 Example context

단독 컴포넌트와 실제 제품 화면 예시를 함께 두어 사용 밀도를 확인합니다.

05 QA evidence

Light/dark, compact density, long text, disabled hover, focus 상태를 시각적으로 확인합니다.
