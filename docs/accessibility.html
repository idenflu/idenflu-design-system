Accessibility

운영 속도를 유지하면서 접근성을 기본값으로 둡니다.

키보드 사용자, 스크린리더 사용자, 저시력 사용자가 같은 상태와 action을 놓치지 않도록 interaction contract를 고정합니다.

Focus

focus-visible은 모든 interactive element에서 보여야 합니다.

Button focus

Focus ring은 primary blue 기반으로 3px ring을 사용합니다.

Input focus는 border와 ring이 함께 바뀌어야 합니다.

Do not hide focus

마우스 hover와 keyboard focus는 같은 상태가 아닙니다. focus-only path를 별도로 검증합니다.

:focus-visible

Keyboard

반복 업무는 키보드만으로 끝낼 수 있어야 합니다.

PatternKeyboard behaviorRequired signal
TabsArrow keys move between tabs, Enter activates.Selected state and panel label
MenuArrow keys move items, Esc closes, Enter selects.Role and active item
ModalFocus is trapped inside, Esc closes when safe.Initial focus and restore target
Table rowRow actions are reachable without hover.Visible action or row menu

Practical flows

대표 패턴은 키보드 흐름까지 문서화합니다.

Modal keyboard flow
  1. Trigger stores restore target.
  2. Dialog opens with title and safe initial focus.
  3. Tab stays inside header, body, footer actions.
  4. Esc or close returns focus to the trigger.
Table row flow
  1. Checkbox selects the row scope.
  2. Primary cell opens the object detail.
  3. Status, owner, and row actions stay reachable.
  4. Overflow menu closes back to its trigger.
Combobox flow
  1. Input announces expanded state.
  2. Arrow keys move the active option.
  3. Enter selects and preserves typed value context.
  4. Esc collapses without losing focus.

Interaction QA

문서 기준은 실제 키보드 동작으로 검증합니다.

Overlay focus trap

Modal이 열리면 Tab은 dialog 안에서 순환하고 Esc 또는 close 후 trigger로 돌아갑니다.

  • Initial focus: safe action or first field
  • Trap: first/last focusable wrap
  • Restore: original trigger
Table keyboard navigation

Review queue row는 hover 없이도 Arrow key와 Tab으로 행 action에 접근할 수 있어야 합니다.

  • ArrowDown/ArrowUp move row focus
  • Home/End move first or last row
  • Row action remains tabbable
Form error linkage

오류 summary, field, helper text가 id로 연결되어 사용자가 바로 복구할 수 있어야 합니다.

  • aria-invalid on field
  • aria-describedby to helper
  • role alert for blocking summary

Focus order

운영자가 읽는 순서대로 조작할 수 있어야 합니다.

Review queue row

Checkbox → campaign link → owner/status controls → row action → overflow 순서로 이동합니다.

  • row 전체를 button으로 만들지 않습니다.
  • hover-only action은 피합니다.
  • overflow menu는 Esc로 닫고 trigger로 focus를 복귀합니다.
Form with errors

Summary alert → 첫 error field → recovery action 순서로 이동합니다.

  • summary link는 field helper id로 연결합니다.
  • submit 후 focus는 summary로 이동합니다.
  • field value는 보존합니다.
Modal decision

Title → reason copy → secondary action → primary/danger action 순서로 이동합니다.

  • 초기 focus는 가장 안전한 control로 둡니다.
  • danger action은 마지막에 배치합니다.
  • 닫힌 뒤 trigger로 focus를 복귀합니다.

Labels

아이콘만 있는 액션은 이름이 있어야 합니다.

Icon button

시각적으로 아이콘만 보여도 accessible name은 구체적인 동사와 대상이어야 합니다.

aria-label="Download report"
Form field

Placeholder는 label이 아닙니다. 항상 visible label을 둡니다.

Status text Due today

상태는 색상만 쓰지 않고 텍스트 label을 함께 제공합니다.
