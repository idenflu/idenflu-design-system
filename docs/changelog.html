Changelog

디자인 시스템 변경을 버전과 영향도로 관리합니다.

토큰, 컴포넌트, 패턴이 바뀔 때 어떤 화면과 구현이 영향을 받는지 추적해 제품 화면의 일관성을 유지합니다.

Current release

Alpha · 2026-06-06.

alpha Enterprise baseline

White light mode, black-base dark mode, 0px radius, compact controls, component page split, visual QA, token export를 기준으로 고정했습니다.

components Component documentation standard

모든 상세 컴포넌트 페이지에 Anatomy, Variants, States, Accessibility, Examples, QA 기준을 추가했습니다.

governance Build and verification flow

공통 topbar/sidebar는 `site.config.json`과 `src/pages`에서 생성하고, 토큰/아이콘/페이지 링크는 검증 스크립트로 확인합니다.

Release impact

QA automation 변경은 문서, 토큰, 인터랙션 영향도를 함께 기록합니다.

QA automation

Browser QA dry run, token usage report, visual QA check를 release checklist에 포함합니다.

Token source

component-token-usage.json으로 컴포넌트별 token 연결을 추적합니다.

Interaction risk

Overlay, table, combobox, tabs keyboard 경로는 별도 scenario로 고정합니다.

Affected components

변경된 token 또는 pattern이 button, input, card, table, overlay 중 어디에 영향을 주는지 먼저 기록합니다.

Required QA

Light/dark, compact density, keyboard path, long text, disabled hover를 release별 필수 확인 항목으로 둡니다.

Deprecations

제거하거나 바꿔야 하는 패턴.

Decorative gradients

브랜드 느낌을 위해 쓰던 장식성 gradient는 제거합니다. 정보 구조, contrast, typography로 premium tone을 만듭니다.

Rounded application chrome

버튼, input, card, chip의 radius는 0px입니다. 예외가 필요하면 component contract에 사유를 남깁니다.

Hidden operational metadata

revision, owner, due date, risk는 hover나 tooltip 뒤로 보내지 않습니다. 본문 row 또는 card body에 노출합니다.

Release checklist

토큰이나 컴포넌트 변경 전 확인합니다.

Token sync

`design.md`, `tokens.json`, `styles.css`의 색/타입/크기 값이 같은 의미로 유지되는지 확인합니다.

Component impact

변경된 token이 button, input, table, feedback, overlay에 어떤 상태 차이를 만드는지 확인합니다.

Visual QA

`visual-qa.html`에서 light/dark, hover, focus, selected, loading, disabled 상태를 한 번에 비교합니다.

Accessibility

Keyboard path, aria-label, aria-sort, aria-invalid, aria-describedby 연결이 깨지지 않았는지 확인합니다.

Docs IA

새 페이지가 topnav, sidebar, page map, Reference group에서 접근 가능한지 확인합니다.

Implementation check

빌드 스크립트와 검증 스크립트를 실행하고 HTML/CSS/JS/JSON/SVG 문법을 함께 확인합니다.
