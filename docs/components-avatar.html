Components

Avatar

owner, reviewer, creator를 텍스트와 함께 식별하는 기준입니다.

Component standard

Avatar 문서의 공통 구조.

Anatomy

Image, initials, name, subtitle, presence, fallback을 나눕니다.

  • Image
  • initials
  • name
  • subtitle
  • presence
Variants

Image avatar, initials, unassigned, stacked group, identity row

States

Image, initials, unassigned, online, busy, disabled 상태를 비교합니다.

Accessibility

정보성 이미지는 alt, 장식 avatar는 hidden, identity row는 name을 visible로 둡니다.

Examples

Owner, requester, approver, collaborator group에 사용합니다.

QA

이미지 실패, 긴 이름, email subtitle, stacked overlap, dark contrast를 확인합니다.

Document template

Avatar 문서는 사람이 누구인지 읽히는 정보 단위로 봅니다.

01 Overview

Avatar만으로 식별하지 말고 name이나 role과 함께 사용합니다.

02 Anatomy

Image, initials, fallback, name, subtitle, presence를 분리합니다.

03 Variants

Single, group, identity row, unassigned, system user를 구분합니다.

04 States

Online, busy, disabled, missing image, fallback initials를 비교합니다.

05 Accessibility

Avatar alt, decorative hidden, visible identity text를 확인합니다.

06 QA

Long name, email wrap, stacked group, dark surface를 검증합니다.

Review checklist
  • Avatar 옆에는 가능한 한 visible name을 함께 둡니다.
  • Fallback initials는 실제 이름에서 가져오고 임의 색상은 만들지 않습니다.
  • Group avatar는 인원 수와 상세 진입 경로를 함께 제공합니다.

Token contract

Avatar는 크기와 identity typography를 일관되게 씁니다.

DecisionTokenRule
Size{sizing.control-height}기본 avatar는 control height와 같은 크기를 사용합니다.
Initials{typography.body-emphasis}Fallback initials는 작지만 선명한 emphasis를 사용합니다.
Surface{colors.surface-2}이미지가 없을 때 neutral surface를 사용합니다.
Presence{colors.primary}Presence는 보조 정보이며 이름을 대체하지 않습니다.

API contract

Avatar API 구현 계약.

TypeContract
Propsname, image, initials, size, presence, subtitle
Statesimage, initials, unassigned, online, busy, disabled
A11yimg alt when informative, decorative avatar hidden, visible name near identity rows

Token usage

Avatar tokens를 실제 token path에 연결합니다.

{sizing.control-height} {typography.body-emphasis} {colors.surface-2} {colors.primary} {colors.hairline}

Usage guidance

idenflu 화면에서 쓰는 기준을 먼저 정합니다.

Use

owner, reviewer, creator처럼 사람이 누구인지 빠르게 식별해야 할 때 씁니다.

Avoid

avatar만으로 사람을 식별하게 만들지 않습니다. 이름이나 역할 없이 이미지에 의존하지 않습니다.

Replace with

책임자 정보는 identity row, 그룹 상태는 avatar group + visible count를 씁니다.
