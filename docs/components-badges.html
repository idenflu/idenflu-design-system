Components

Badges

상태, count, filter chip을 짧고 직접적으로 표시하는 기준입니다.

Component standard

Badges 문서의 공통 구조.

Anatomy

Tone, label, count, optional icon, remove action을 나눕니다.

  • Text label
  • tone
  • count
  • remove button
  • status meaning
Variants

Status badge, count badge, filter chip, removable tag, risk label

States

Neutral, info, success, warning, danger, selected, disabled 상태를 비교합니다.

Accessibility

상태는 색상만으로 전달하지 않고 remove action은 aria-label을 사용합니다.

Examples

테이블 상태, 필터 적용값, owner label, asset risk에 사용합니다.

QA

긴 label, count overflow, disabled chip, dark contrast를 확인합니다.

Context example

Badges는 상태와 적용된 조건을 작은 단위로 보여줍니다.

Selected Review pending Approved Disclosure risk
Review checklist
  • Status badge는 상태 text를 반드시 포함합니다.
  • Filter chip은 선택 조건과 reset 가능 여부를 드러냅니다.
  • Count badge는 단독 의미가 없으면 label 옆에 붙입니다.

Token contract

Badges는 caption과 semantic accent를 작은 면적으로 사용합니다.

DecisionTokenRule
Text{typography.caption}Badge label은 12px caption으로 유지합니다.
Info{colors.primary-soft}선택과 정보는 primary soft를 사용합니다.
Success{colors.accent-mint}완료 상태는 mint와 완료 label을 함께 씁니다.
Warning{colors.accent-amber}대기와 위험 예고는 amber와 원인 text를 함께 둡니다.

API contract

Badge API 구현 계약.

TypeContract
Propstone, label, count, removable, status
Statesneutral, info, success, warning, danger, selected, disabled
A11ytext visible, status not color-only, remove button aria-label

Token usage

Badge tokens를 실제 token path에 연결합니다.

{typography.caption} {colors.primary-soft} {colors.accent-mint} {colors.accent-amber} {radius.none}

Usage guidance

idenflu 화면에서 쓰는 기준을 먼저 정합니다.

Use

review, locked, ready, selected count처럼 짧은 상태나 범주를 본문 옆에 붙일 때 씁니다.

Avoid

상태의 이유나 다음 action을 badge 안에만 담지 않습니다.

Replace with

설명이 필요하면 feedback, 여러 조건 선택은 filter chip, 중요한 action은 button을 씁니다.
