Components

Banner

페이지 안에 머무르는 inline 메시지로 info, success, warning, error 톤과 선택적 action을 전달하는 기준입니다.

Component standard

Banner 문서의 공통 구조.

Anatomy

Icon, tone, title, message, action, dismiss 자리를 나눕니다.

  • tone icon
  • title
  • message
  • action
  • dismiss
Variants

Info, success, warning, error 톤과 dismissible/persistent 구성을 구분합니다.

States

Info, success, warning, error, dismissed 상태를 비교합니다.

Accessibility

낮은 심각도는 role=status, error는 role=alert로 알리고 색상에만 의존하지 않습니다.

Examples

동기화 안내, 승인 마감, 권한 제한, 시스템 오류 복구에 사용합니다.

QA

긴 메시지 wrap, dark contrast, reduced motion, dismiss 포커스 복귀를 확인합니다.

Document template

Banner 문서는 톤과 dismiss 여부, live region을 먼저 정합니다.

Context example

Banner는 화면 흐름을 막지 않고 맥락 안에서 상태와 다음 action을 알립니다.

i
예약된 동기화

다음 데이터 동기화는 18:00에 시작됩니다.

!
승인 마감 임박

2건의 검토가 오늘 마감입니다.

Review checklist
  • 톤은 색상만이 아니라 아이콘과 text로 함께 구분합니다.
  • error와 같이 즉시 알릴 메시지는 role=alert로 전달합니다.
  • 닫을 수 있는 banner는 dismiss action에 aria-label을 둡니다.

Token contract

Banner는 surface 위에 semantic accent를 낮은 대비로 사용합니다.

DecisionTokenRule
Text{typography.body-sm}메시지는 body small을 기준으로 둡니다.
Info{colors.primary-soft}정보 톤은 primary soft 배경을 사용합니다.
Warning{colors.accent-amber}주의 톤은 amber 강조와 원인 text를 함께 둡니다.
Error{colors.accent-coral}오류 톤은 coral 강조와 복구 action을 함께 둡니다.

API contract

Banner API 구현 계약.

TypeContract
Propstone, title, message, action, dismissible
Statesinfo, success, warning, error, dismissed
A11yrole status for low severity, role alert for error, severity not color-only, dismiss button aria-label

Token usage

Banner tokens를 실제 token path에 연결합니다.

{typography.body-sm} {colors.primary-soft} {colors.accent-amber} {colors.accent-coral} {colors.hairline}

Usage guidance

idenflu 화면에서 쓰는 기준을 먼저 정합니다.

Use

페이지나 섹션 범위의 상태, 안내, 복구 메시지를 흐름 안에 계속 남겨야 할 때 씁니다.

Avoid

색상만으로 심각도를 전달하거나 짧게 사라져야 할 알림을 banner로 고정하지 않습니다.

Replace with

잠깐 뜨는 결과는 toast, 빈 화면 안내는 empty state, 필드 오류는 input helper를 씁니다.

Banner

네 가지 톤을 같은 구조로 정렬하고 심각도에 맞는 live region을 사용합니다.

i
Info

진행에 영향이 없는 안내는 role=status로 조용히 전달합니다.

v
Success

완료 결과는 mint 톤과 완료 text를 함께 보여줍니다.

!
Warning

대기나 주의가 필요한 상태는 원인 text를 함께 둡니다.

x
Error

차단 상태는 role=alert로 즉시 알리고 복구 action을 제공합니다.
