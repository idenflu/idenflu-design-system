Components

Breadcrumb

계층 구조 안에서 현재 위치와 상위 경로를 짧게 보여주는 navigation trail 기준입니다.

Component standard

Breadcrumb 문서의 공통 구조.

Anatomy

Root link, intermediate link, separator, current item, overflow 자리를 나눕니다.

  • root link
  • separator
  • intermediate links
  • current item
  • overflow ellipsis
Variants

Default trail, truncated trail, single-level, icon-leading 경로를 구분합니다.

States

Default, hover, focus, current page, truncated, disabled 상태를 비교합니다.

Accessibility

nav landmark와 aria-label을 두고 현재 위치는 aria-current=page, 구분자는 aria-hidden 처리합니다.

Examples

캠페인 상세, 설정 하위 페이지, 긴 폴더 경로 이동에 사용합니다.

QA

긴 label, 좁은 폭 truncation, dark contrast, focus ring을 확인합니다.

React: <Breadcrumb items={[...]} /> — data-driven <nav aria-label="Breadcrumb"> trail. 마지막(또는 current) 항목은 aria-current="page" span, 나머지는 href 링크이며 구분자는 aria-hidden입니다.

Review checklist
  • Breadcrumb는 nav landmark와 aria-label을 함께 가집니다.
  • 현재 페이지는 link가 아니라 aria-current="page"로 표시합니다.
  • 경로가 길면 가운데를 접고 양 끝 위치를 유지합니다.

Token contract

Breadcrumb는 낮은 대비의 link와 약한 구분자를 사용합니다.

DecisionTokenRule
Link{colors.primary}이동 가능한 상위 경로는 primary link 색을 사용합니다.
Text{colors.ink-muted}비활성/기본 경로 text는 ink-muted로 낮춥니다.
Separator{colors.hairline}구분자는 hairline 수준의 약한 대비로 둡니다.
Density{density.compact}trail은 compact density 높이를 따릅니다.

API contract

Breadcrumb API 구현 계약.

TypeContract
Propsitems, separator, current, maxItems, overflow
Statesdefault, hover, focus, current page, truncated, disabled
A11ynav landmark with aria-label, aria-current page for current item, decorative separator aria-hidden, keyboard focus order follows trail

Token usage

Breadcrumb tokens를 실제 token path에 연결합니다.

{density.compact} {typography.body-sm} {colors.primary} {colors.hairline} {colors.ink-muted}

Usage guidance

idenflu 화면에서 쓰는 기준을 먼저 정합니다.

Use

사용자가 깊은 계층(워크스페이스, 캠페인, 상세) 안에서 현재 위치와 상위 경로로 빠르게 이동해야 할 때 씁니다.

Avoid

단계가 1~2단계로 얕거나 순차 진행이 의미인 화면을 breadcrumb로 표현하지 않습니다. 현재 페이지를 다시 link로 만들지 않습니다.

Replace with

순서 진행은 stepper, 같은 화면의 view 전환은 tabs, page 단위 이동은 pagination 또는 navigation을 씁니다.
