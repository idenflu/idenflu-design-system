Chips

선택된 값, 필터, tag처럼 사용자가 조작하는 작은 값 단위 기준입니다.

Component standard

Chips 문서의 공통 구조.

Anatomy

Label, value, selected state, remove action, group label을 나눕니다.

  • label
  • value
  • selected state
  • remove action
  • group label
Variants

Filter chip, tag, selected value, removable chip, disabled chip

States

Default, hover, focus, selected, disabled, removing 상태를 비교합니다.

Accessibility

Interactive chip role, remove aria-label, selected text, group label을 확인합니다.

Examples

Active filters, selected creators, label tags처럼 값 목록 조작에 사용합니다.

QA

Long label, remove focus, wrapping, disabled contrast를 확인합니다.

Token contract

Chips는 compact control token과 selected surface를 함께 씁니다.

DecisionTokenRule
Height{sizing.control-height}Interactive chip은 compact control 높이를 따릅니다.
Text{typography.caption}긴 label은 wrap 또는 truncation 기준을 명시합니다.
Selected{colors.primary-soft}Selected chip은 badge status와 다른 선택 surface를 사용합니다.
Remove{spacing.xs}Remove icon과 label 사이 간격을 유지합니다.

API contract

Chip API 구현 계약.

TypeContract
Propslabel, value, tone, selected, removable
Statesdefault, hover, focus, selected, disabled, removing
A11ybutton when interactive, remove button aria-label, selected text visible, group label

Token usage

Chip tokens를 실제 token path에 연결합니다.

{typography.caption} {sizing.control-height} {colors.primary-soft} {colors.hairline} {spacing.xs}

Usage guidance

idenflu 화면에서 쓰는 기준을 먼저 정합니다.

Use

필터 값, 선택된 tag, removable metadata처럼 사용자가 값 목록을 조작해야 할 때 씁니다.

Avoid

상태를 chip 색상만으로 표현하거나 primary action처럼 보이게 만들지 않습니다.

Replace with

상태 표시는 badge, 옵션 선택은 controls/select, 보조 명령 묶음은 menu를 씁니다.
