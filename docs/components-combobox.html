Combobox

긴 목록에서 검색하고 선택하는 owner, creator, label 입력 기준입니다.

Component standard

Combobox 문서의 공통 구조.

Anatomy

Label, input, popup, option, active option, selection을 나눕니다.

  • Visible label
  • query input
  • option list
  • active option
  • helper copy
Variants

Single select, multi select, async search, owner picker, tag picker

States

Collapsed, expanded, typing, active, empty, loading, disabled 상태를 비교합니다.

Accessibility

role combobox, aria-controls, aria-activedescendant, option keyboard path를 확인합니다.

Examples

Owner, campaign, label, saved view처럼 검색 가능한 선택에 사용합니다.

QA

긴 option, empty result, loading, active option, mobile keyboard를 확인합니다.

Token contract

Combobox는 field와 menu surface 토큰을 함께 씁니다.

DecisionTokenRule
Input{sizing.field-height}입력 영역은 field와 같은 높이를 사용합니다.
Popup{colors.surface-raised}옵션 영역은 surface-raised와 hairline으로 구분합니다.
Active{colors.primary-soft}키보드 active option은 selected와 구분되는 soft surface를 씁니다.
Focus{colors.primary}입력 focus와 option active가 동시에 읽히도록 유지합니다.

Usage guidance

idenflu 화면에서 쓰는 기준을 먼저 정합니다.

Use

owner, creator, label처럼 목록이 길고 검색 후 하나를 선택해야 할 때 씁니다.

Avoid

선택지가 적거나 사용자가 값을 직접 입력해야 하는 필드를 combobox로 만들지 않습니다.

Replace with

선택지가 적으면 native select/controls, 자유 입력이면 input, 다중 조건이면 search toolbar를 씁니다.
