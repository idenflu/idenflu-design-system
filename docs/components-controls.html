Controls

현재 값이 유지되는 필터, 설정, 선택 control 기준입니다.

Component standard

Controls 문서의 공통 구조.

Anatomy

기본 구성요소를 정리해 구현과 디자인 리뷰의 출발점을 맞춥니다.

  • Label
  • selected state
  • control group
  • helper copy
  • disabled state
Variants

Segmented control, switch, checkbox, range, density toggle

States

Default, hover, focus, selected, loading, disabled, error 또는 locked 상태를 같은 density로 비교합니다.

Accessibility

Keyboard path, visible focus, semantic role, accessible name, helper/error 연결을 함께 검토합니다.

Examples

단독 컴포넌트보다 idenflu 제품 화면 안에서 쓰이는 예시를 우선합니다.

QA

Visual QA에서 light/dark, compact density, disabled hover, focus, long text overflow를 다시 확인합니다.

Document template

Controls 문서는 선택 범위와 상태 전환을 먼저 고정합니다.

01 Overview

컨트롤이 필터, 설정, 상태 변경 중 어떤 역할인지 정의합니다.

02 Anatomy

Label, current value, control target, helper, disabled reason을 나눕니다.

Checked, selected, indeterminate, disabled, focus 상태를 비교합니다.

05 Accessibility

Native input, group label, aria-checked, keyboard path를 확인합니다.

06 QA

Long label, compact density, disabled hover, range value 표시를 검증합니다.

Context example

컨트롤은 테이블과 설정 화면에서 선택 범위를 명확하게 만듭니다.

Review filter controlsCompact
Review checklist
  • 선택값이 하나면 segmented control, 여러 개면 checkbox group을 사용합니다.
  • Switch는 즉시 적용되는 설정에만 사용합니다.
  • Range는 현재 값을 숫자로 항상 함께 보여줍니다.

Token contract

컨트롤은 compact density와 focus token을 기준으로 맞춥니다.

DecisionTokenRule
Height{sizing.control-height}기본 컨트롤은 버튼과 같은 높이 체계를 씁니다.
Compact{density.compact}테이블 toolbar 안에서는 compact 높이와 8px gap을 사용합니다.
Selected{colors.primary-soft}선택 상태는 soft background와 primary border로 표시합니다.
Focus{colors.primary}키보드 focus는 hover와 별개로 항상 보입니다.

API contract

Control API 구현 계약.

TypeContract
Propslabel, value, checked, group, helper
Stateschecked, selected, indeterminate, disabled, focus
A11ynative input, group label, aria-checked where needed, keyboard path

Token usage

Control tokens를 실제 token path에 연결합니다.

{sizing.control-height} {density.compact} {colors.primary-soft} {colors.primary}

Usage guidance

idenflu 화면에서 쓰는 기준을 먼저 정합니다.

Use

필터, on/off 설정, 다중 선택처럼 현재 값이 화면에 계속 남아야 할 때 씁니다.

Avoid

즉시 실행 명령을 switch나 checkbox로 표현하지 않습니다.

Replace with

일회성 명령은 button, 여러 명령 묶음은 menu, 모드 전환은 tabs를 씁니다.
