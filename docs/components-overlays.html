Usage guidance

idenflu 화면에서 쓰는 기준을 먼저 정합니다.

Use

현재 화면 맥락은 유지하되 승인, 반려, 상세 확인 같은 결정 표면을 분리해야 할 때 씁니다.

Avoid

간단한 inline 수정이나 반복 row action을 모두 modal로 열지 않습니다.

Replace with

가벼운 보조 정보는 tooltip/popover, 행 단위 명령은 menu, 페이지 전환이 필요한 상세는 navigation을 씁니다.

Overlay decision guide

Overlay를 열기 전에 더 낮은 표면으로 해결 가능한지 확인합니다.

01 Use inline

필드 helper, validation, 권한 사유처럼 사용자가 바로 고쳐야 하는 정보는 inline으로 둡니다.

02 Use disclosure

audit, blocker detail, advanced setting처럼 보조 상세는 같은 화면에서 접고 펼칩니다.

03 Use menu

행 주변의 보조 명령이나 정렬/필터는 menu로 낮춥니다.

04 Use popover

작은 선택이나 확인이 trigger 근처에서 끝나면 backdrop 없는 popover를 씁니다.

05 Use modal

삭제, 반려, 권한 변경처럼 현재 화면을 잠시 멈추고 결정해야 할 때만 modal을 씁니다.

NeedSurfaceReason
Explain field errorInline helper오류는 사용자가 값을 고치는 자리 옆에 있어야 합니다.
Show audit detailDisclosure맥락은 유지하고 상세만 열면 됩니다.
Choose row actionMenu반복 행 명령은 화면을 막을 필요가 없습니다.
Confirm destructive actionModal결정 전 impact와 reason을 확인해야 합니다.

Overlay states

Open 상태만 만들지 말고 pending, validation, locked까지 정의합니다.

Pending action

Assign, save, reject처럼 네트워크가 필요한 action은 버튼 label과 loading state를 유지합니다.

  • aria-busy 사용
  • 중복 submit 방지
  • 취소 가능 여부를 명확히 표현
Validation

Popover나 modal 안의 field error는 overlay 안에서 해결되어야 합니다.

  • 첫 error로 focus 이동
  • aria-describedby 연결
  • 값 보존 후 retry
Locked overlay

권한이 없어 action이 불가능하면 dialog를 열기 전에 disabled + reason을 우선합니다.

  • 권한 요청 action 제공
  • owner 또는 policy 노출
  • 숨겨진 destructive action은 피합니다

Focus restore

닫힌 뒤 사용자가 있던 자리로 돌아와야 합니다.

Open
  1. Trigger stores restore target.
  2. aria-expanded becomes true.
  3. Overlay becomes visible.
  4. First safe focus target receives focus.
Navigate
  1. Tab order stays inside modal or drawer.
  2. Esc closes when it is non-destructive.
  3. Menu arrow keys move active menuitem.
  4. Validation sends focus to the first invalid field.
Close
  1. Overlay hidden state is restored.
  2. aria-expanded becomes false.
  3. Focus returns to trigger.
  4. Selection/filter context remains unchanged.
