Pagination

테이블과 리스트에서 현재 위치와 데이터 범위를 보여주는 page 이동 기준입니다.

Component standard

Pagination 문서의 공통 구조.

Anatomy

Nav landmark, page buttons, current page, page size, item count를 나눕니다.

  • nav landmark
  • page buttons
  • current page
  • page size
  • item count
Variants

Compact, full, table footer, mobile summary, page size selector

States

First page, middle page, last page, loading, disabled, compact 상태를 비교합니다.

Accessibility

aria-current page, button label, disabled state, nav label을 확인합니다.

QA

Token contract

Pagination은 compact button과 current surface token을 씁니다.

DecisionTokenRule
Button{sizing.control-height}Page button은 compact control 높이를 유지합니다.
Current{colors.primary}현재 page는 aria-current와 primary 색상으로 표시합니다.
Border{colors.hairline}Footer 경계와 button border는 hairline을 씁니다.
Text{typography.body-sm}Item count와 page size는 body-sm로 표시합니다.

API contract

Pagination API 구현 계약.

TypeContract

Statesfirst page, middle page, last page, loading, disabled, compact

Token usage

Pagination tokens를 실제 token path에 연결합니다.

{sizing.control-height} {typography.body-sm} {colors.surface-raised} {colors.hairline} {colors.primary}

Usage guidance

idenflu 화면에서 쓰는 기준을 먼저 정합니다.

Use

테이블이나 리스트가 여러 페이지로 나뉘고 현재 위치와 page size가 반복 조작될 때 씁니다.

Avoid

무한 스크롤 화면이나 짧은 리스트에 pagination을 붙이지 않습니다.

Replace with

짧은 결과는 count summary, 연속 탐색은 load more, 큰 데이터 필터링은 search toolbar를 씁니다.

Pagination

테이블 footer와 search result에서 같은 기준을 씁니다.
