Progress

저장, 업로드, 동기화, 로딩 상태를 사용자가 판단할 수 있게 보여주는 기준입니다.

Component standard

Progress 문서의 공통 구조.

Anatomy

Label, status text, indicator, busy region, completion state를 나눕니다.

  • label
  • status text
  • indicator
  • busy region
  • completion
Variants

Progress bar, skeleton, inline spinner, upload progress, step progress

States

Idle, indeterminate, determinate, skeleton, complete, error 상태를 비교합니다.

Accessibility

role progressbar, aria-valuenow, aria-busy, status text를 확인합니다.

Examples

Upload, saving, syncing, list loading처럼 시간이 걸리는 상태에 사용합니다.

QA

Skeleton contrast, progress text, reduced motion, error recovery를 확인합니다.

Token contract

Progress는 surface, primary, status token을 함께 씁니다.

Usage guidance

idenflu 화면에서 쓰는 기준을 먼저 정합니다.

Use

업로드, 저장, 동기화, 데이터 로딩처럼 진행 중임을 사용자가 판단해야 할 때 씁니다.

Avoid

즉시 끝나는 action에 skeleton이나 progress bar를 과하게 사용하지 않습니다.

Replace with

짧은 action은 button loading, 결과 없음은 empty state, 오류는 feedback을 씁니다.
