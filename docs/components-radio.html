Radio

여러 옵션 중 정확히 하나만 고르는 단일 선택 그룹 기준입니다.

Component standard

Radio 문서의 공통 구조.

Anatomy

Fieldset, legend, radio circle, dot, label을 나눕니다.

  • fieldset
  • legend
  • radio circle
  • selected dot
  • label
Variants

Vertical group, horizontal group, with description를 구분합니다.

States

Unselected, selected, focus, disabled, error 상태를 비교합니다.

Accessibility

Native radio, fieldset/legend group label, arrow key 이동, 그룹당 단일 선택을 지킵니다.

Examples

Approval routing, review density, export format 선택에 사용합니다.

QA

긴 label, disabled option, focus ring, dark contrast를 확인합니다.

Context example

Radio는 상호 배타적인 선택지를 모두 보여줍니다.

Approval routing
Review checklist
  • Radio는 정확히 하나만 선택할 때 사용합니다.
  • 옵션이 모두 한눈에 보여야 할 때만 씁니다.
  • fieldset/legend로 그룹 라벨을 항상 제공합니다.

Token contract

Radio는 control height와 primary accent를 사용합니다.

DecisionTokenRule
Size{sizing.control-height}Radio 정렬 높이는 control-height 체계에 맞춥니다.
Selected{colors.primary}선택된 radio는 primary border와 dot으로 표시합니다.
Unselected{colors.hairline}선택되지 않은 circle은 hairline border를 씁니다.
Surface{colors.surface-2}Radio circle 배경은 surface-2를 씁니다.

Usage guidance

idenflu 화면에서 쓰는 기준을 먼저 정합니다.

Use

상호 배타적인 옵션 2~5개를 모두 화면에 보여주고 정확히 하나만 선택하게 할 때 씁니다.

Avoid

옵션이 많아 목록이 길어지거나 다중 선택이 필요할 때는 radio를 쓰지 않습니다.

Replace with

옵션이 많으면 select 또는 combobox, 다중 선택은 checkbox, 즉시 적용 on/off는 switch, 모드 전환은 segmented control 또는 tabs를 씁니다.

Radio

그룹 안에서 하나의 옵션만 선택합니다.

Radio group

Approval routing

여러 옵션 중 정확히 하나만 선택하고 옵션이 모두 보여야 할 때 사용합니다. React: <RadioGroup label="Approval routing" options={[...]} value onChange /> — data-driven options에 fieldset/legend 그룹 라벨, orientation vertical/horizontal을 지원합니다.

Horizontal

Review density

옵션이 두세 개로 짧으면 orientation="horizontal"로 한 줄에 배치합니다. 네이티브 <input type="radio">와 화살표 키 이동을 그대로 사용합니다.
