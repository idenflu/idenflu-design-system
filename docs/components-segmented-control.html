Segmented control

서로 배타적인 view/mode를 inline에서 하나만 고르는 전환 기준입니다.

Component standard

Segmented control 문서의 공통 구조.

Anatomy

Track, segment buttons, active segment, label을 나눕니다.

  • track
  • segment buttons
  • active segment
  • group label
Variants

2-option, 3-option, compact density를 구분합니다.

States

Default, hover, focus, selected, disabled 상태를 비교합니다.

Accessibility

role=group, group label, aria-pressed, 화살표 키보드 이동을 지킵니다.

Examples

대시보드 기간, 채널, 리뷰 상태 전환에 사용합니다.

QA

긴 label, compact density, dark contrast, focus ring을 확인합니다.

Context example

Segmented control은 한 번에 하나의 view만 선택된다는 점을 분명히 합니다.

Review checklist
  • 옵션은 2~4개로 제한하고 모두 한 줄에 보이게 합니다.
  • 항상 정확히 하나의 옵션만 active 상태로 둡니다.
  • 옵션이 많거나 길면 select 또는 tabs로 바꿉니다.

Token contract

Segmented control은 control height와 primary accent를 사용합니다.

DecisionTokenRule
Height{sizing.control-height}Track 높이는 control-height 기준으로 버튼과 맞춥니다.
Compact{density.compact}테이블 toolbar 안에서는 compact 높이와 gap을 사용합니다.
Selected{colors.primary-soft}선택된 segment는 soft background와 primary border로 표시합니다.
Focus{colors.primary}키보드 focus는 hover와 별개로 항상 보입니다.

API contract

Segmented control API 구현 계약.

TypeContract
Propslabel, options, value, onChange, size, disabled
Statesdefault, selected, hover, focus, disabled
A11yrole=group, aria-pressed on active option, single-select only, keyboard arrow navigation

Token usage

Segmented control tokens를 실제 token path에 연결합니다.

{sizing.control-height} {density.compact} {colors.primary-soft} {colors.primary} {colors.hairline}

Usage guidance

idenflu 화면에서 쓰는 기준을 먼저 정합니다.

Use

서로 배타적인 view/mode 2~4개를 inline에서 즉시 전환하고 현재 선택이 한눈에 보여야 할 때 씁니다.

Avoid

옵션이 5개 이상이거나 라벨이 길거나 다중 선택이 필요할 때는 segmented control을 쓰지 않습니다.

Replace with

옵션이 많으면 select, 페이지 단위 전환은 tabs, 다중 선택은 checkbox, 즉시 on/off는 switch를 씁니다.
