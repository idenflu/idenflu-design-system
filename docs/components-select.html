Token contract

Select는 field 토큰을 따르고 상태 색상만 분리합니다.

DecisionTokenRule
Height{sizing.field-height}Native select는 input과 같은 40px field 높이를 사용합니다.
Text{typography.body}Option label은 본문 크기를 유지하고 helper는 body-sm로 낮춥니다.
Border{colors.hairline}기본과 disabled는 hairline으로 유지하고 focus에서만 강조합니다.
Focus / Error{colors.primary} / {colors.accent-coral}Focus는 primary, invalid는 error text와 border로 분리합니다.

Usage guidance

idenflu 화면에서 쓰는 기준을 먼저 정합니다.

Use

캠페인 단계, reviewer 역할, timezone, density처럼 짧고 고정된 단일 선택지에 씁니다.

Avoid

긴 owner/creator 목록이나 검색이 필요한 선택을 select로 만들지 않습니다. 비활성 option의 이유를 숨기지 않습니다.

Replace with

목록이 길거나 검색이 필요하면 combobox, 다중 조건은 controls/search toolbar, 직접 입력은 input을 씁니다.

Select

선택지는 짧게 유지하고 결과 상태를 바로 노출합니다.

React: <Select label="Review stage" options={[...]} placeholder="Select stage" error helperText required /> — 네이티브 <select>를 label·helper·error와 연결하고 SelectOption.disabled로 비활성 옵션을 표시합니다. expanded·multiple·searchable 또는 SelectOption.icon/SelectOption.description를 주면 native control 대신 커스텀 role="listbox" 폼으로 전환합니다. 아래 데모의 요약 패널은 문서용 인터랙션이며, 컴포넌트는 정적 helperText/error만 노출합니다.

Expanded select

아이콘·설명·다중 선택·검색이 필요하면 expanded listbox 폼으로 전환합니다.

Native <select>는 짧고 단순한 단일 선택에 쓰고, 옵션에 아이콘·보조 설명이 필요하거나 다중 선택·검색이 필요하면 role="listbox" 팝업으로 바꿉니다. 아래 데모는 React <Select>의 expanded 계약을 문서용 vanilla JS로 재현합니다.
