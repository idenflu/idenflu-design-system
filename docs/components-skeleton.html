Skeleton

데이터가 로딩되는 동안 콘텐츠 형태를 미리 보여 주는 placeholder 기준입니다. Spinner나 progress bar와 달리 최종 레이아웃의 자리를 그대로 유지합니다.

Component standard

Skeleton 문서의 공통 구조.

Anatomy

Block, line, shimmer, label 자리를 나눕니다.

  • block
  • line
  • shimmer
  • label slot
Variants

Text line, card, table row, avatar placeholder를 구분합니다.

States

Loading, shimmer, static, resolved 상태를 비교합니다.

Accessibility

aria-busy로 로딩을 알리고 장식 요소는 aria-hidden 처리합니다.

Examples

테이블, 카드, 상세 패널 로딩에 사용합니다.

QA

긴 로딩, reduced motion, dark contrast를 확인합니다.

Review checklist
  • Skeleton 블록은 실제 콘텐츠와 같은 자리와 개수를 차지합니다.
  • 데이터가 준비되면 즉시 실제 콘텐츠로 교체하고 placeholder를 남기지 않습니다.
  • 장식 블록은 aria-hidden으로 숨기고 옆에 visible Loading 문구를 둡니다.
  • reduced motion 환경에서는 shimmer를 멈추고 정적 면으로 보여 줍니다.

Token contract

Skeleton은 surface와 hairline을 낮은 대비로 사용합니다.

DecisionTokenRule
Surface{colors.surface-2}Placeholder 면은 surface-2를 사용합니다.
Edge{colors.hairline}블록 경계는 hairline으로 약하게 둡니다.
Radius{radius.md}블록 모서리는 radius md를 따릅니다.
Motion{motion.duration.base}shimmer는 base duration으로 반복합니다.

API contract

Skeleton API 구현 계약.

TypeContract
Propsvariant, lines, width, rounded, animated
Statesloading, shimmer, static, done
A11yaria-busy true, aria-hidden decorative, announce when resolved

Token usage

Skeleton tokens를 실제 token path에 연결합니다.

idenflu 화면에서 쓰는 기준을 먼저 정합니다.

Avoid
