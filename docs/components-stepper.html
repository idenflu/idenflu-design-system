Review checklist
  • 현재 step은 aria-current와 visible label을 함께 씁니다.
  • Blocked step은 원인과 다음 action을 제공합니다.
  • Optional step은 건너뛸 수 있음을 text로 표시합니다.

Token contract

Stepper는 진행 상태를 색상과 텍스트로 함께 표현합니다.

DecisionTokenRule
Label{typography.body-emphasis}Step label은 강조 텍스트로 스캔되게 합니다.
Current{colors.primary}현재 단계는 primary와 aria-current를 함께 씁니다.
Complete{colors.accent-mint}완료 단계는 mint와 완료 text를 함께 둡니다.
Divider{colors.hairline}단계 연결선은 hairline 수준으로만 표시합니다.

Usage guidance

idenflu 화면에서 쓰는 기준을 먼저 정합니다.

Use

brief, review, approval처럼 순서와 현재 단계, 막힌 이유가 중요한 workflow에 씁니다.

Avoid

단순 상태 badge나 탭 전환을 stepper로 과장하지 않습니다.

Replace with

화면 모드 전환은 tabs, 독립 상태 요약은 badges, 상세 복구는 feedback을 씁니다.
