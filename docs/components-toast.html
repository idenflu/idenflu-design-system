Review checklist
  • 필수 오류는 toast가 아니라 field helper나 feedback에 둡니다.
  • Undo가 있으면 toast 안에 명확한 action을 제공합니다.
  • Dismiss button은 accessible name을 갖습니다.

Token contract

Toast는 raised surface, shadow, status tone을 함께 씁니다.

DecisionTokenRule
Surface{colors.surface-raised}Toast는 화면 위에 뜨지만 overlay처럼 맥락을 막지 않습니다.
Shadow{shadow.panel}Stack surface는 panel shadow로 떠 있는 계층을 표현합니다.
Success{colors.accent-mint}성공 tone은 짧은 결과 알림에만 씁니다.
Error{colors.accent-coral}오류 toast는 복구 action이나 상세 링크를 함께 둡니다.

Usage guidance

idenflu 화면에서 쓰는 기준을 먼저 정합니다.

Use

저장 완료, 동기화 실패, undo 가능한 단기 결과처럼 화면 흐름을 막지 않는 알림에 씁니다.

Avoid

필수 오류, 권한 이유, 장기 진행 상태를 toast에만 숨기지 않습니다.

Replace with

필드 오류는 helper, 화면 상태는 feedback, 결정이 필요한 내용은 overlay를 씁니다.

Toast
