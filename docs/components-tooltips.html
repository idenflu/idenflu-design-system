Component standard

Tooltips 문서의 공통 구조.

Anatomy

Trigger, hint, placement, optional title, close path를 나눕니다.

  • Trigger
  • tooltip
  • popover surface
  • placement
  • close path
Variants

Tooltip, metadata popover, quick action popover, disabled hint

States

Hidden, visible, hover, focus, pinned, disabled 상태를 비교합니다.

Accessibility

aria-describedby, keyboard trigger, focusable popover close path를 확인합니다.

Examples

Icon button hint, metadata detail, disabled action reason에 사용합니다.

QA

긴 hint, viewport edge, focus reveal, dark mode contrast를 확인합니다.

Document template

Tooltips 문서는 설명과 상호작용 popover를 분리합니다.

Hint, metadata, disabled reason, quick action popover를 구분합니다.

05 Accessibility

aria-describedby, trigger focus, popover close, critical action 금지를 확인합니다.

06 QA

Viewport edge, long content, dark contrast, keyboard reveal을 검증합니다.

Review checklist
  • Tooltip 안에는 중요한 결정이나 form을 넣지 않습니다.
  • Popover가 interactive하면 닫기 경로와 focus 순서를 제공합니다.
  • Disabled action reason은 tooltip보다 visible helper를 우선합니다.

Token contract

Tooltips는 inverse surface와 caption으로 짧게 유지합니다.

DecisionTokenRule
Tooltip surface{colors.inverse-canvas}짧은 tooltip은 inverse surface를 사용해 주변 UI와 구분합니다.
Tooltip text{typography.caption}힌트는 caption 크기로 1-2줄 안에 유지합니다.
Popover surface{colors.surface-raised}상호작용 popover는 일반 surface와 hairline을 씁니다.
Spacing{spacing.xs}Trigger와 tooltip 간격은 작게 유지해 소속을 드러냅니다.

API contract

Tooltip API 구현 계약.

TypeContract
Propstrigger, content, placement, interactive, delay
Stateshidden, visible, hover, focus, pinned, disabled

Token usage

Tooltip tokens를 실제 token path에 연결합니다.

{typography.caption} {colors.inverse-canvas} {colors.inverse-ink} {shadow.low} {spacing.xs}

Usage guidance

idenflu 화면에서 쓰는 기준을 먼저 정합니다.

Use

이미 보이는 UI의 짧은 보조 설명이나 약어 해석을 제공할 때 씁니다.

Avoid

필수 오류, 권한 이유, 주요 metadata를 tooltip 안에만 숨기지 않습니다.

Replace with

결정에 필요한 정보는 inline helper, feedback, popover 또는 detail panel로 올립니다.
