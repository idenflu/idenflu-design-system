Foundations

White canvas, crisp action, visible state.

idenflu는 명확한 white canvas를 기본으로 쓰고, 필요할 때 black-base dark mode로 전환해 장시간 운영 화면에서도 같은 정보 밀도를 유지합니다.

Semantic token aliases

상태 의미는 색 이름이 아니라 역할 이름으로 먼저 정합니다.

컴포넌트는 mint, amber, coral을 직접 설명하지 않고 status-success, status-warning, status-error, focus-ring 같은 의미 alias로 사용합니다.

status-success Approved, live, synced

완료되었거나 정상 흐름으로 돌아온 상태입니다. Badge, toast, progress complete에서 사용합니다.

status-warning Pending, deadline risk

아직 실패는 아니지만 사용자가 확인해야 하는 상태입니다.

status-error Blocked, failed, invalid

저장 실패, 필드 오류, 승인 차단처럼 복구 action이 필요한 상태입니다.

focus-ring Keyboard-visible focus

Button, select, disclosure, pagination, overlay trigger에서 같은 focus affordance를 씁니다.

Token pipeline

토큰 변경은 JSON에서 CSS 변수까지 같은 흐름으로 관리합니다.

Source

`tokens.json`이 machine-readable token source입니다. 색, theme, spacing, sizing, radius, shadow, density, icon registry를 포함합니다.

Build

아래 명령이 `tokens.generated.css`를 생성합니다.

node scripts/build-tokens.js
Verify

전체 사이트 검증에서 token CSS 링크, token 값, 기존 CSS 변수 동기화를 확인합니다.

node scripts/verify-site.js
Change typeRequired filesCommandRisk
Color tokendesign.md, tokens.jsonbuild-tokens + build-site + verify-siteLight/dark contrast regression
Spacing/sizingtokens.json, styles.css if layout changesbuild-tokens + visual-qa-checkMobile overflow or control height mismatch
Icon registrytokens.json, icons.svg, components-icons.htmlverify-siteBroken symbol reference or missing accessible name

Motion

모션은 세 단계 duration과 두 개의 easing으로만 제어합니다.

Duration은 표면 크기에 비례해 올라가고, easing은 기본적으로 standard를 쓰며 강조가 필요할 때만 emphasized를 선택합니다. 임의의 ms 값이나 ease 곡선을 직접 쓰지 않습니다.

TokenValueUseRule
--if-duration-fast120msHover, press, focus ring즉각적인 micro-interaction 피드백에 씁니다.
--if-duration-base200msToggle, dropdown, accordion표준 상태 전환의 기본값입니다.
--if-duration-slow320msModal, drawer, page surface큰 표면이 들어오고 나갈 때만 씁니다.
--if-ease-standardcubic-bezier(0.2, 0, 0, 1)대부분의 enter/exit 전환자연스럽게 감속하는 기본 easing입니다.
--if-ease-emphasizedcubic-bezier(0.3, 0, 0, 1.4)주의를 끄는 강조 모션살짝 overshoot해 강조가 필요할 때만 씁니다.

Layering

z-index는 base에서 toast까지 토큰 사다리로만 쌓습니다.

Overlay 스택은 base, dropdown, sticky, overlay, modal, popover, toast 순서로 쌓입니다. backdrop(overlay)은 modal surface 바로 아래 층이고, popover와 tooltip은 modal 안에서 그 위로 올라오며, toast는 항상 최상단의 비차단 알림 층입니다.

LayerTokenz-indexUsed by
Base--if-z-base0Page content, table — 양수 z-index를 직접 쓰지 않습니다.
Dropdown--if-z-dropdown1000Select, menu 등 trigger에 앵커된 목록.
Sticky--if-z-sticky1100Sticky toolbar, table header.
Overlay--if-z-overlay1200Modal backdrop layer.
Modal--if-z-modal1300Modal, drawer surface (backdrop 위).
Popover--if-z-popover1400Modal 안에서 열리는 popover, tooltip.
Toast--if-z-toast1500항상 최상단의 비차단 알림.

Breakpoint

반응형 분기는 sm/md/lg/xl 네 개의 기준 폭으로 맞춥니다.

CSS 변수는 @media 쿼리 조건에서 사용할 수 없으므로, 아래 토큰과 같은 값을 build/SCSS 단계 상수로 함께 유지하고 미디어 쿼리에는 raw px 값을 적습니다.

BreakpointTokenMin widthTarget
sm--if-breakpoint-sm640pxLarge phone — compact single-column layout 시작점.
md--if-breakpoint-md768pxTablet portrait — two-column으로 전환합니다.
lg--if-breakpoint-lg1024pxSmall laptop — sidebar와 content를 함께 배치합니다.
xl--if-breakpoint-xl1280pxDesktop — full dashboard density.
