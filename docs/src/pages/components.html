Components

컴포넌트 카탈로그는 인덱스로 유지하고, 세부 문서는 컴포넌트별 페이지에서 관리합니다.

Component catalog

컴포넌트별 상세 문서로 분리했습니다.

각 페이지는 Overview, variants, states, accessibility, examples를 확장하기 위한 독립 문서입니다.

Component API

핵심 컴포넌트는 props, states, accessibility 계약을 함께 봅니다.

component-api.json을 기준으로 Button, Input, Table, Overlay 문서의 구현 계약을 한곳에서 비교합니다.

Component detail template

모든 상세 페이지는 같은 깊이로 판단합니다.

컴포넌트는 예쁜 샘플보다 구현 계약, 상태, 접근성, 실제 화면 검증까지 함께 문서화되어야 합니다.

01 Required depth

Anatomy, variants, states, accessibility, examples, QA를 같은 순서로 확인합니다.

02 State model

Default, hover, focus, selected, loading, disabled, error, locked를 빠뜨리지 않습니다.

03 A11y contract

Keyboard path, accessible name, semantic role, focus restore를 구현 전 기준으로 둡니다.

04 Example context

단독 컴포넌트와 실제 제품 화면 예시를 함께 두어 사용 밀도를 확인합니다.

05 QA evidence

Light/dark, compact density, long text, disabled hover, focus 상태를 시각적으로 확인합니다.
