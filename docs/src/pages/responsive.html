Responsive

좁은 화면에서도 운영 정보와 action을 잃지 않습니다.

idenflu의 responsive 기준은 장식적 재배치가 아니라 table, toolbar, sidebar, long text가 실제 업무에서 계속 작동하게 하는 운영 규칙입니다.

Shell behavior

Navigation은 줄어도 현재 위치는 유지합니다.

Sidebar collapse

Desktop sidebar는 tablet 이하에서 horizontal section nav로 바뀝니다.

  • 현재 page group label은 유지합니다.
  • active item은 primary-soft로 유지합니다.
  • Reference group은 primary topnav를 늘리지 않습니다.
Topbar overflow

Topbar는 brand, active section, theme/action만 남기고 부가 metadata는 숨깁니다.

  • 핵심 navigation은 horizontal scroll 허용
  • 긴 label은 줄임보다 짧은 label 재작성 우선
  • CTA는 본문 action area로 이동 가능
Action footer

승인, 저장, 반려처럼 중요한 mobile action은 화면 하단에 고정할 수 있습니다.

  • Primary 1개 + quiet cancel
  • Danger action은 확인 modal에서 처리
  • Keyboard focus 순서는 본문 뒤에 위치
