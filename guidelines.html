Usage Guidelines

컴포넌트 선택 기준을 운영 흐름에 맞게 정리합니다.

같은 화면을 여러 사람이 만들어도 primary action, validation, table density, bulk action이 같은 원칙으로 판단되도록 문서화합니다.

Component contracts

기본 컴포넌트의 구현 계약.

Button contract

하나의 decision area에서 action priority와 state가 일관되게 읽히도록 정리합니다.

AnatomyLabel, optional icon, fixed size, accessible name StatesDefault, hover, focus, selected, loading, disabled, danger KeyboardEnter/Space activates; disabled is not interactive ARIAIcon-only uses aria-label; loading uses aria-busy
Input contract

입력은 label, value, helper/error copy가 같은 field 안에서 함께 읽혀야 합니다.

AnatomyVisible label, control, helper text, optional prefix/suffix StatesDefault, focus, success, invalid, readonly, disabled KeyboardTab enters field; text editing stays native ARIAErrors use aria-invalid and aria-describedby
Select contract

상태나 owner처럼 값이 제한된 항목은 native select 또는 combobox로 구분합니다.

AnatomyLabel, selected value, option set, optional leading icon StatesDefault, focus, open, selected, disabled, invalid KeyboardNative select keeps platform keyboard behavior ARIASearchable selection uses combobox/listbox/option
Table contract

운영 메타데이터는 숨기지 않고 비교 가능한 컬럼으로 노출합니다.

AnatomyToolbar, header, rows, selection bar, row actions, pagination StatesSorted, selected, loading, empty, filtered, permission locked KeyboardCheckboxes, headers, row actions stay reachable ARIAUse table roles, column headers, aria-sort where active
Tabs contract

탭은 화면 모드 전환에만 사용하고 선택 상태와 panel 연결을 명확히 둡니다.

AnatomyTablist, tabs, active tab, labelled tabpanel StatesDefault, hover, focus, selected, disabled when needed KeyboardArrow keys move; Enter/Space activates ARIArole tablist/tab/tabpanel, aria-controls, aria-selected
Feedback contract

상태는 색만으로 읽히지 않게 원인과 다음 action을 함께 제공합니다.

AnatomyStatus label, title, reason, recovery action StatesInfo, success, warning, error, loading, empty KeyboardRecovery actions stay in logical Tab order ARIAImportant async updates use polite live regions

Do / Don't

제품 화면에서 바로 적용할 판단 기준.

AreaDoDon't
Actionsdecision area마다 primary 1개, 보조는 quiet/ghost로 낮춥니다.같은 영역에 filled button을 반복해서 화면 소음을 만들지 않습니다.
Dataowner, status, revision, deadline, risk를 본문 컬럼으로 보여줍니다.핵심 운영 메타데이터를 tooltip, hover menu, sort menu 뒤에 숨기지 않습니다.
Formslabel, input, helper/error, prefix/suffix를 하나의 field 단위로 묶습니다.placeholder만으로 field 의미를 전달하지 않습니다.
Stateempty, loading, error, locked 상태에서도 다음 action을 제공합니다.상태를 색 점 하나나 아이콘 하나만으로 표현하지 않습니다.

Density system

운영 화면의 정보 밀도를 의도적으로 선택합니다.

Compact

테이블, 툴바, 반복 row처럼 하루에도 여러 번 보는 화면에 사용합니다.

  • Control 32px
  • Row 40-44px
  • Gap 8-12px
Comfortable

일반 dashboard, form, detail panel의 기본 density입니다.

  • Control 38px
  • Field 40px
  • Gap 12-16px
Spacious

중요한 승인, 설정 변경, 오류 복구처럼 판단 시간이 필요한 화면에만 사용합니다.

  • Control 44px
  • Panel padding 24px
  • Gap 20-24px

Primary

승인, 생성, 저장처럼 화면의 다음 결정이 명확할 때만 사용합니다.

Filter creators

Forms

입력은 값, 상태, 조치가 함께 보여야 합니다.

기본 입력은 label을 항상 노출하고 placeholder를 label 대체로 쓰지 않습니다.

에러는 색만 쓰지 않고 해결 가능한 helper copy를 함께 제공합니다.

disabled는 조작 불가, readonly는 복사 가능한 고정값으로 구분합니다.
