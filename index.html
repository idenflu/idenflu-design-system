System positioning

특정 화면이 아니라 idenflu의 제품 판단 방식을 정의합니다.

예시 데이터는 실제 제품 맥락을 보여주기 위한 재료이고, 시스템 기준은 구조, 상태, 밀도, 접근성, 의사결정 흐름입니다.

System lens

색, 타입, 간격, 컴포넌트는 idenflu가 정보를 어떻게 읽히게 하는지 설명합니다.

Example context

캠페인, 리뷰, 권한 같은 예시는 실제 데이터 밀도와 상태 판단을 검증하기 위한 샘플입니다.

Decision surface

화면은 사용자가 현재 상태와 다음 action을 빠르게 판단하도록 구성합니다.

Principles

새 화면을 시작할 때 먼저 보는 간단한 원칙.

01

Metadata first

상태, 기한, 오너, 리비전처럼 판단에 필요한 정보를 먼저 보이게 둡니다.

02

One clear path

주요 action은 하나의 흐름으로 읽히게 두고, 보조 action은 조용하게 둡니다.

03

Visible state

상태는 색만으로 설명하지 않고 텍스트와 다음 action을 함께 둡니다.

04

Compact, readable

운영 화면의 밀도는 유지하되 label, row, action은 충분히 읽히게 만듭니다.

05

Plain structure

장식보다 surface, hairline, spacing으로 화면의 구조를 분명하게 만듭니다.

06

idenflu context

브랜드의 제품 맥락과 실제 데이터가 범용 관리 화면보다 먼저 느껴지게 합니다.

Shape

카드, 버튼, input, chip 모두 radius를 제거하고 hairline으로만 구분합니다.

Recommended path

처음 보는 사람도 같은 순서로 판단하게 합니다.

1 Foundations

색, 타입, spacing, density의 기본 언어를 확인합니다.

2 Guidelines

action, state, data view, overlay 선택 기준을 먼저 맞춥니다.

3 Components

컴포넌트별 anatomy, variants, states, a11y, QA 기준을 적용합니다.

4 Patterns

화면 조합으로 확장하고 실제 제품 흐름 안에서 검증합니다.

5 Visual QA

light/dark, focus, disabled, long text, responsive 회귀를 확인합니다.
