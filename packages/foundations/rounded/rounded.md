# Rounded (Border Radius)

모서리 둥글기(`Rounded`)는 시스템 전반에 일관된 인상을 형성하고 인터페이스의 시각적 성격을 조정하기 위한 기준입니다.

`Rounded`는 단순한 장식 요소가 아니라 컴포넌트의 밀도, 강조 수준, 상호작용 특성을 전달하기 위한 **시각적 언어**로 사용됩니다.
기본적으로 시스템은 직선적이고 구조적인 형태를 유지하며, 필요한 경우에만 `Rounded`를 적용하여 **요소의 성격과 중요도를 표현**합니다.

## Philosophy (철학 및 원칙)

1. **형태를 통한 위계 표현**  
    모서리 둥글기는 컴포넌트가 담고 있는 정보의 단위와 시각적 무게에 비례하여 적용됩니다.
   - 작은 UI(배지 · 툴팁)  
     사용자 시선을 강하게 끌거나 독립된 제어 역할을 하므로, 상대적으로 크고 뚜렷한 반경을 적용하여 부듣러운 인상을 제공
   - 대형 컨테이너(페이지 · 모달)  
     화면 전체의 구조를 단단하게 잡아주어야 하므로, 작은 반경을 사용하여 시각적 안정감과 견고함을 확보

2. **체계적인 확장**
   모든 `Rounded` 값은 공통 스케일을 기반으로 구성되며, 제품 전반에서 일관된 시각 경험을 제공합니다.

## Core Concepts (핵심 개념)

1. **기본값**  
   `rounded-none`은 `Rounded`의 기본 값입니다.  
   특별한 시각적 의도나 컨테이너 구조를 가질 때 상하 단계로 확장합니다.

2. **원형**  
   `rounded-full`은 유동적인 크기를 가진 요소(예: 태그, 뱃지 등)의 양 끝을 완벽한 반원으로 처리하기 위해 물리적으로 매우 큰 값(`9999px`)을 부여하는 값입니다.

## Usage Guidelines (사용 기준 및 가이드라인)

### Do (권장 사항)

- 요소가 작을수록 더 큰 반경을 사용
- 텍스트 길이에 따라 좌우로 늘어나는 컴포넌트(뱃지, 태그)에는 `rounded-full`을 사용하여 일관된 모양을 유지

### Dont't (금지 사항)

- 임의의 값을 사용하지 않기
- 대형 컨테이너에 과도하게 큰 값을 사용하지 않기
- 동일한 컴포넌트 계열에서 여러 값을 혼용하지 않기

## Tokens & Values (토큰 및 값)

| Token        | Value  |
| ------------ | ------ |
| rounded-none | 0px    |
| rounded-sm   | 2px    |
| rounded-md   | 4px    |
| rounded-lg   | 8px    |
| rounded-xl   | 12px   |
| rounded-full | 9999px |
