Buttons

Action hierarchy, size, icon, loading, disabled states.

Component standard

Buttons 문서의 공통 구조.

Anatomy

기본 구성요소를 정리해 구현과 디자인 리뷰의 출발점을 맞춥니다.

  • Label
  • optional icon
  • fixed size
  • loading indicator
  • disabled reason
Variants

Primary, secondary, quiet, ghost, danger, icon-only, sizes

States

Default, hover, focus, selected, loading, disabled, error 또는 locked 상태를 같은 density로 비교합니다.

Accessibility

Keyboard path, visible focus, semantic role, accessible name, helper/error 연결을 함께 검토합니다.

Examples

단독 컴포넌트보다 campaign, creator, review, admin workflow 안에서 쓰이는 예시를 우선합니다.

QA

Visual QA에서 light/dark, compact density, disabled hover, focus, long text overflow를 다시 확인합니다.

Button spec

구현자가 바로 참조할 수 있는 상세 계약.

PropertyValuesRuleEdge case
variantprimary, secondary, quiet, ghost, dangerdecision area마다 primary는 1개만 둡니다.row 반복 액션은 ghost 또는 icon-only로 낮춥니다.
sizesmall, medium, largesmall 32px, medium 38px, large 44px 높이를 고정합니다.긴 label은 wrap하지 않고 action label을 짧게 다시 씁니다.
iconleading, none, icon-only텍스트 버튼 icon은 leading만 사용합니다.icon-only는 aria-label을 함께 둡니다.
loadingtrue, falsearia-busy와 inline indicator를 함께 사용하고 label은 유지합니다.loading 중 중복 제출은 막고 width shift를 만들지 않습니다.
disabledtrue, falsedisabled는 hover, active, focus style이 살아나면 안 됩니다.승인 불가처럼 이유가 필요한 경우 주변 helper copy를 둡니다.
StateVisual contractAccessibility contract
FocusPrimary blue border + 3px focus ringTab 접근 순서가 시각 순서와 일치해야 합니다.
SelectedPrimary soft background + primary border토글성 버튼이면 aria-pressed 또는 선택 상태 문맥을 제공합니다.
DangerError fill + explicit destructive label삭제/거절을 icon-only로 단독 표현하지 않습니다.
