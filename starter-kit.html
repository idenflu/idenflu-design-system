Starter kit

새 화면은 가볍게 시작하고, 필요한 만큼만 깊게 들어갑니다.

이 페이지는 상세 규칙을 먼저 읽기보다 workflow, layout, component 조합, 상태 체크를 빠르게 선택하는 출발점입니다.

Start here

처음에는 네 가지만 정합니다.

1. Workflow

사용자가 처리할 대상, 상태, 권한, 다음 action을 먼저 고릅니다.

2. Layout

비교가 많으면 list/table, 판단이 많으면 detail/form, 짧은 결정은 overlay로 시작합니다.

3. Components

Button, input, table, feedback처럼 이미 있는 기본 조합을 먼저 사용합니다.

4. States

Loading, empty, error, locked, success 중 화면에 필요한 상태만 골라 둡니다.

Principles

세부 규칙보다 먼저 보는 6가지 원칙.

Metadata first

상태, owner, due, revision은 화면 판단을 돕는 기본 정보입니다.

One clear path

사용자가 다음에 할 일을 한 가지 중심으로 읽을 수 있게 둡니다.

Visible state

상태는 색보다 텍스트와 action으로 이해되게 합니다.

Compact, readable

운영 화면은 촘촘해도 읽을 수 있어야 합니다.

Plain structure

불필요한 장식보다 hairline, spacing, hierarchy로 구분합니다.

Flexible depth

처음에는 간단히 만들고, 실제 workflow가 보일 때 상세 규칙을 더합니다.

Layout recipes

완성 화면이 아니라 뼈대부터 고릅니다.

List + detail

비교 후 상세를 여는 목록, 테이블, 큐 화면.

Form + preview

입력과 확인이 함께 필요한 설정, 작성, 초대 흐름.

Dashboard band

요약 지표, 파이프라인, 최근 activity를 빠르게 스캔하는 화면.

Decision overlay

승인, 반려, 권한 요청처럼 현재 맥락 안에서 짧게 끝나는 결정.
