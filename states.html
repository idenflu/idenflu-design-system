States

상태는 색이 아니라 운영 맥락입니다.

idenflu의 상태 체계는 사용자가 현재 위치, 원인, 다음 행동을 한눈에 판단하도록 loading, empty, error, success, locked를 같은 밀도로 보여줍니다.

State model

모든 상태는 원인과 다음 action을 포함합니다.

Informational

동기화, 필터 적용, 검토 대기처럼 사용자가 흐름을 이해해야 할 때 사용합니다.

Info
Success

승인, 전송, 저장처럼 작업 결과가 확정되었을 때 짧게 표시합니다.

Approved
Warning

기한 임박, 누락된 briefing, budget drift처럼 아직 복구 가능한 상태입니다.

Due today
Error

실패 원인과 복구 action을 함께 표시합니다. 색만으로 판단하게 하지 않습니다.

Failed
Disabled

권한이나 워크플로우상 조작이 막힌 상태입니다. hover나 active 스타일을 살리지 않습니다.

Readonly

값은 복사하거나 읽을 수 있지만 편집할 수 없습니다. disabled보다 텍스트 가독성을 높입니다.

Rev 03 · Locked

Loading

불확실한 대기보다 예상 가능한 자리 유지.

Skeleton row

리스트와 테이블은 최종 레이아웃 자리를 유지합니다.

Action loading

버튼 내부 loading은 label을 유지하고 중복 클릭을 막습니다.

Background sync Syncing 4 assets

사용자가 계속 작업할 수 있으면 blocking overlay를 열지 않습니다.

Empty

빈 화면은 다음 행동을 제안합니다.

No creators matched

필터를 줄이거나 신규 creator를 초대하세요.

No assets uploaded

캠페인 briefing 후 첫 asset을 업로드합니다.

Review queue clear

승인 대기 asset이 없습니다. scheduled campaign을 확인하세요.

Error

에러는 원인, 영향, 복구를 순서대로 보여줍니다.

Upload failed

파일 크기가 200MB 제한을 초과했습니다.

Deadline risk

2개 asset의 검토 기한이 오늘 18:00입니다.

Saved

Brief changes are visible to assigned creators.

Permission

권한 잠금은 기능 부재가 아니라 정책 상태입니다.

Approval locked

Brief owner만 asset approval을 완료할 수 있습니다.

Archived campaign

종료된 campaign은 읽기 전용으로 유지됩니다.

readonly
Billing required

Invoice owner에게 접근 권한을 요청하세요.
