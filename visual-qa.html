Visual QA

컴포넌트 상태를 한 화면에서 비교합니다.

버튼, 입력, 테이블, 상태 컴포넌트가 light/dark와 hover/focus/disabled 조건에서 같은 밀도와 대비를 유지하는지 확인합니다.

01 Component state QA

Button, input, table, overlay의 default, hover, focus, selected, loading, disabled 상태를 같은 화면에서 비교합니다.

02 Dark mode deep QA

Black-base dark mode에서 text, hairline, focus ring, selected surface, disabled contrast가 흐려지지 않는지 확인합니다.

03 Release evidence

토큰, 컴포넌트, 문서 IA 변경은 release checklist와 changelog impact에 함께 기록합니다.

Coverage matrix

QA 범위를 케이스별로 분리합니다.

각 범위는 자동화 명령, 수동 확인 항목, 대표 컴포넌트를 함께 연결합니다.

01 Actions

Button, icon button, destructive action, loading, disabled hover를 같은 기준으로 확인합니다.

  • Primary는 결정 영역에서 하나만 남습니다.
  • Icon-only action은 aria-label을 가집니다.
  • Disabled 상태는 hover와 active style이 살아나지 않습니다.
node scripts/visual-qa-check.js
02 Forms

Input, combobox, date time, validation helper가 label과 상태를 유지하는지 확인합니다.

  • Label, helper, error는 field 안에서 연결됩니다.
  • Combobox는 active option과 keyboard state를 노출합니다.
  • Read-only와 disabled는 서로 다른 시각 상태를 가집니다.
node scripts/browser-qa-check.js --dry-run
03 Data

Table, search toolbar, filter chip, attachment row가 dense layout에서도 정보를 잃지 않는지 확인합니다.

  • Row action과 revision 정보가 작은 폭에서도 유지됩니다.
  • Sort, selection, expanded row의 aria 상태를 검토합니다.
  • 검색과 필터 해제 action이 같은 toolbar 안에 남습니다.
node scripts/component-coverage-check.js
04 Surfaces

Card, overlay, menu, tooltip, popover의 header, body, footer와 focus 흐름을 확인합니다.

  • Overlay backdrop은 뒤 화면을 약하게 흐리게 합니다.
  • Modal, drawer, popover는 케이스별 예시로 분리됩니다.
  • Card footer action은 하단에 고정된 구조를 유지합니다.
node scripts/verify-site.js
05 Feedback

Badge, empty state, progress, toast, locked state가 원인과 다음 action을 같이 제공하는지 확인합니다.

  • 상태는 색상만으로 전달하지 않습니다.
  • Empty와 error는 recovery action을 함께 제공합니다.
  • Permission state는 제한 이유를 visible copy로 설명합니다.
node scripts/visual-qa-check.js
06 Themes

Light, black-base dark, compact density에서 token contrast와 focus ring을 비교합니다.

  • Text, hairline, selected surface의 contrast를 확인합니다.
  • Focus ring은 양쪽 theme에서 선명하게 보입니다.
  • Token usage는 tokens.json 경로와 일치합니다.
node scripts/token-usage-report.js --check

Regression checklist

토큰이나 상태 스타일을 바꾸기 전 확인할 항목.

Theme contrast

Light는 white canvas에서, dark는 black canvas에서 text, hairline, focus가 충분히 보이는지 확인합니다.

  • Body text
  • Muted metadata
  • Selected surface
Interactive states

Hover sample, focus ring, selected, loading, disabled가 서로 다른 의미로 읽히는지 비교합니다.

  • Disabled hover leak 없음
  • Loading text 유지
  • Focus visible
Form states

Default, decorated, success, invalid, readonly, disabled가 같은 field anatomy 안에서 정렬되는지 확인합니다.

  • Label visible
  • Error helper connected
  • Prefix/suffix aligned
Data density

Compact table에서도 owner, revision, due, risk, row action이 사라지지 않는지 확인합니다.

  • Bulk bar visible
  • Row action reachable
  • Empty recovery present
Keyboard path

버튼, icon button, tabs, combobox, table action이 시각 순서대로 Tab 접근되는지 확인합니다.

  • Icon button aria-label
  • Combobox active option
  • Modal focus restore
Copy and recovery

Error, empty, locked 상태가 원인과 다음 action을 같이 제공하는지 확인합니다.

  • Filtered empty 분리
  • Permission reason 표시
  • Retry preserves context

Automation

정적 검증으로 상태 계약 누락을 먼저 잡습니다.

Browser QA dry run node scripts/browser-qa-check.js --dry-run

Screenshot/interaction scenario가 현재 HTML과 연결되는지 확인합니다.

Token usage check node scripts/token-usage-report.js --check

컴포넌트별 token path가 tokens.json에 실제로 존재하는지 확인합니다.

Full static QA node scripts/verify-site.js

생성 페이지, token sync, 문서 구조, 금지 문구를 함께 확인합니다.
